# Changelog

## 2.2.0

- Added a mandatory React-ready static architecture while preserving HTML + Tailwind + vanilla JavaScript as the only prototype stack.
- Added stable `data-component`, `data-variant`, `data-state`, and `data-action` contracts.
- Added props/data, slots, callbacks, route, and migration-risk documentation requirements.
- Added separation rules for mock data, storage/services, and DOM controllers.
- Prohibited inline event handlers, behavior bound to Tailwind utility selectors, and complete component injection through `innerHTML`.
- Added `react-readiness.md`, a React component-map template, route-map and handoff deliverables, and an incremental migration sequence.
- Expanded the audit and completion checklist with React-readiness checks.


## 2.1.0

- Added evidence ladder and explicit separation of research findings from assumptions.
- Added proto-persona integrity rules; the Skill must not fabricate validated personas or user findings.
- Added reference-DNA intake with take/reject synthesis and anti-cloning rules.
- Added happy, failure, and difficult/assisted journey mapping.
- Added focused usability-test planning and validation templates.
- Added severity- and confidence-based UX/design review reporting.
- Updated default accessibility target to WCAG 2.2 AA with manual verification requirements.
- Added interface content design, vocabulary, error recovery, onboarding, and localization stress tests.
- Added component state/accessibility specifications and shared motion-token governance.
- Expanded the static audit with semantic, form, accessible-name, duplicate-ID, dialog, SVG, and safe-link checks plus JSON output.
