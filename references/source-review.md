# External Skill Review Notes

This file records ideas adapted into Product UI Design Studio without making any external skill a runtime dependency.

## UI/UX Design Pro

Useful themes incorporated:

- product-design role spanning information architecture, visual hierarchy, interaction flows, personas, usability testing, and design systems
- iterative feedback instead of treating the first prototype as final

Not imported:

- reliance on Material Design or Apple HIG as default visual systems
- framework-specific implementation assumptions

## Claude Marketplaces Design/UI category sample

Useful patterns incorporated:

- evidence-aware UX research and proto-personas
- reference-site DNA extraction with explicit take/reject notes
- design-system master rules plus per-feature/page specifications
- component state matrices and motion tokens
- structured WCAG, keyboard, and screen-reader review
- Nielsen-style heuristic review with severity and confidence
- purposeful UI writing and localization stress testing
- happy, failure, and difficult journey variants

Not imported:

- closed aesthetic taxonomies
- mandatory premium-font assumptions
- arbitrary bans on specific fonts, easings, shadows, or layout forms
- React, Next.js, Vue, mobile-native, or framework component patterns
- claims that automated tools prove accessibility compliance

The user's direction and the product's actual subject remain higher priority than every external reference.
