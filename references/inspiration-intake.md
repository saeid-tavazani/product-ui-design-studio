# Inspiration Intake and Design DNA

Use this reference when the user supplies URLs, screenshots, products, moodboards, or named visual references.

## Rule

References are raw material, not templates. Extract principles and synthesize across them. Never clone a page structure, distinctive composition, illustration, or branded asset.

## Per-reference DNA record

Capture:

- reference and relevant screen or area
- what the user explicitly likes or dislikes
- typography: role, scale, width, rhythm, hierarchy
- color: contrast structure, saturation, semantic use
- geometry: radius, line, shape, icon language
- spacing: density, rhythm, whitespace behavior
- composition: grid, alignment, asymmetry, focal sequence
- imagery: medium, crop, lighting, art direction
- motion: purpose, timing, choreography, restraint
- content: voice, labels, information density
- interaction: navigation model, disclosure, feedback
- **take:** reusable principles appropriate to this product
- **reject:** elements that conflict with product, accessibility, originality, or user preference
- **reason:** why the decision transfers or does not transfer

## Synthesis

After reviewing all references:

1. identify repeated principles
2. identify conflicts
3. connect selected principles to the product subject and audience
4. create one new design thesis and signature element
5. record provenance in `design-profile.json`
6. run a similarity check: the output must not be recognizable as a copy of one reference

A visual reference does not outrank an explicit user instruction.
